# Как проверить статус деплоя на Netlify

## 1. Проверка в Netlify Dashboard

1. Войдите на [Netlify Dashboard](https://app.netlify.com)
2. Откройте ваш сайт **Assignment3-vue**
3. Перейдите на вкладку **"Deploys"**
4. Проверьте последний деплой:
   - ✅ **Published** - деплой успешен
   - ⏳ **Building** - идет сборка
   - ❌ **Failed** - ошибка сборки

## 2. Проверка логов сборки

В разделе **"Deploys"**:
1. Кликните на последний деплой
2. Откройте **"Deploy log"**
3. Проверьте, что:
   - `npm run build` выполнился успешно
   - Нет ошибок компиляции
   - Файлы собраны в `dist/`

## 3. Проверка переменных окружения

1. Перейдите в **Site settings** → **Environment variables**
2. Убедитесь, что установлены:
   - `VITE_API_BASE_URL`
   - `VITE_BACKEND_URL`

## 4. Очистка кэша и пересборка

Если изменения не применились:

1. В **Deploys** нажмите **"Trigger deploy"** → **"Clear cache and deploy site"**
2. Или в **Site settings** → **Build & deploy** → **Clear build cache**

## 5. Проверка собранных файлов

После успешной сборки проверьте:
1. Откройте DevTools (F12) → Network
2. Обновите страницу (Ctrl+Shift+R или Cmd+Shift+R для жесткой перезагрузки)
3. Найдите файл `index-*.js`
4. Откройте его и поищите `getFileUrl` - функция должна быть там

## 6. Проверка в браузере

1. Откройте сайт: https://assignment3-vue.netlify.app
2. Откройте DevTools (F12) → Console
3. Выполните жесткую перезагрузку: **Ctrl+Shift+R** (Windows/Linux) или **Cmd+Shift+R** (Mac)
4. Проверьте, исчезла ли ошибка

## 7. Если ошибка все еще есть

Проверьте:
- Переменные окружения установлены правильно
- Кэш браузера очищен
- Используется последняя версия сайта (проверьте версию в Network tab)
